# ProwlrBot Brand Voice Guidelines

## Tagline

**"Always watching. Always ready."**

Use the tagline in:
- Social media bios and profile descriptions
- Release announcements (as a closing line or header)
- README headers and landing page hero sections

Do **not** use the tagline:
- In every single post (reserve it for key moments)
- Modified or paraphrased (keep it exact)
- In casual community replies or support threads

## Tone Principles

### Professional but Approachable
Write like a knowledgeable colleague, not a corporate press release. Technical accuracy matters, but so does readability. Avoid jargon when a simpler word works.

**Do:** "ProwlrBot now supports Telegram channels — set it up in under a minute."
**Don't:** "We are pleased to announce the general availability of Telegram channel integration capabilities."

### Confident, Not Arrogant
State what the product does well without disparaging alternatives. Let features speak for themselves.

**Do:** "ProwlrBot handles multi-channel orchestration out of the box."
**Don't:** "Unlike inferior tools, ProwlrBot actually works."

### Developer-First
The primary audience is developers and technical operators. Respect their intelligence. Include concrete details — version numbers, config snippets, CLI commands — over vague promises.

**Do:** "Run `prowlr app` and you're live on port 8088."
**Don't:** "Getting started is super easy and takes no time at all!"

### Community-Oriented
Acknowledge contributors, celebrate community wins, and invite participation. Open source thrives on inclusion.

**Do:** "Thanks to @contributor for adding Discord support in #529!"
**Don't:** "Our team has built Discord support."

## Messaging Hierarchy

When writing about ProwlrBot, prioritize these themes in order:

1. **Autonomy** — Agents that work independently, monitor continuously, and act on your behalf
2. **Multi-channel** — One agent, every platform (Discord, Telegram, Slack, DingTalk, Feishu, iMessage, and more)
3. **Extensibility** — Skills, MCP tools, custom channels, provider-agnostic model support
4. **Privacy** — Self-hosted, local models supported, secrets encrypted at rest
5. **Open Source** — Community-driven, transparent, forkable

## Vocabulary

| Use | Avoid |
|-----|-------|
| ProwlrBot | Prowlr Bot, prowlrbot (in prose), PROWLRBOT |
| agent | bot (except in the product name) |
| skill | plugin, extension |
| channel | integration, connector |
| provider | vendor, supplier |
| `prowlr` (monospace) | prowlr (in prose without code formatting) |

## Hashtags (Social Media)

Primary: `#ProwlrBot` `#AIAgent` `#OpenSource`
Secondary: `#Automation` `#SelfHosted` `#DevTools` `#LLM`

Use 2-4 hashtags per post. Place them at the end of the post, not inline.

## Emoji Usage

Use sparingly and purposefully:
- Announcements and releases: one emoji at the start is fine
- Technical documentation: no emojis
- Community posts: match the platform culture (more on Discord, fewer on HN)
